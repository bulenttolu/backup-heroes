3 Yöntem · 3 Farklı Yaklaşım

Aynı SQL workload'ı, farklı katmanlarda korunuyor

1

Ajansız İmaj Yedekleme (VM Tabanlı)

Yönetim: AltyapıVSS: Gerekli
  • Sanal makine seviyesinde çalışır, ekstra bir ajan kurmaya gerek yok
  • SQL veritabanının uygulama tutarlı yedeği alınır
  • Arzu edilirse transaction log'ları da alınır ve truncate edilir
Ne zaman: Sanal makine ortamlarında.
2

Veeam Agent for Windows

Yönetim: AltyapıVSS: Gerekli
  • Fiziksel sunucularda ve SQL cluster yapılarında çalışır
  • İmaj ile beraber uygulama tutarlı yedek alır
  • Protection Groups ile birden fazla ajanı tek noktadan yönetirsiniz
Ne zaman: Fiziksel sunucular, Microsoft Failover Cluster için gerekli.
3

Veeam Plug-in for SQL Server

Yönetim: DBAVSS: Gerekmiyor
  • SQL Server içinden native T-SQL komutlarıyla tetiklenir (full / diff / log)
  • SSMS (SQL Server Management Studio) içinden yönetilir
  • Veeam Backup & Replication'da kataloglama ve raporlama sağlanır
Ne zaman: Kendi DBA ekibi olan ve native araçlardan vazgeçmek istemeyen yapılar için.
