PostgreSQL Instance Nasıl Bulunur?

Backup job başladığında agent'ın discovery mantığı şu soruyla başlar: XML dosyası var mı? /etc/veeam/VeeamPostgreSQLAgent.xml

Hayır — XML yok

Sadece Varsayılan Dizinler Taranır

  • /etc/postgresql
  • /var/lib/postgresql
  • /var/lib/pgsql

Bu dizinlerin dışındaki bir instance bulunamaz.

Evet — XML var

AddConfigDirs ile Custom Path Eklenir

<config AddConfigDirs="/home/user/pgconfdir/"/>

Instance bulunur, pgsqlagent devam eder.

Not: XML dosyası oluşturulmazsa, custom path'teki instance'lar backup'a dahil edilmez.
Veeam Explorer for PostgreSQL

Kurtarma İçin 4 Yöntem

01 · INSTANCE

Data Restore

Instance'ın tüm data directory ve WAL dosyalarını orijinal veya farklı bir konuma kopyalar, ardından instance'ı başlatır. Sadece instance seviyesinde çalışır.

02 · INSTANCE

Data Publishing

Backup'ı FUSE ile geçici olarak mount edip instance'ı doğrudan hedef sunucuya bağlar; tam kopyalama olmadan pgAdmin gibi araçlarla hızlı erişim sağlar.

03 · INSTANCE

Instant Recovery

Instance'ı doğrudan backup'tan anında çalıştırır, arka planda pg_basebackup ile hedef sunucuya senkronize eder ve senkronizasyon bitince switchover yapılır.

04 · VERİTABANI

Data Export

Veritabanlarını taşınabilir DUMP dosyaları olarak export eder; pg_restore ile farklı bir PostgreSQL sunucusuna geri yüklenebilir. Veritabanı seviyesinde çalışan tek yöntemdir.

Not: Point-in-time kurtarma için PostgreSQL WAL seviyesinin replica veya üstü olması gerekir.
