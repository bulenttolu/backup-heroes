Hangi ana geri dönülecek? (her yöntemde geçerli)
01

Mevcut Nokta

Backup'taki restore point anına döner. En basit, en hızlı senaryo.

02

Point-in-Time

Full/Bulk-logged model + log backup şart; loglar hedef sunucuda replay edilir.

03

Belirli Transaction

Transaction listesi staging sunucuda gösterilir. En yavaş ama en hassas yöntem.

Hedef: orijinal konum ya da alternatif sunucu — her yöntemde seçilebilir.
AlwaysOn AG notu: log replay / transaction restore staging sunucuda gerçekleşir; staging sürümü kaynaktan eski olmamalı.

Bağlama / Restore Düzeyi
01

Publish Database

  • Backup'tan doğrudan attach, restore adımı yok
  • Hızlı ama geçici; Explorer kapanınca ayrılır
02

Restore Database

  • Orijinal/alternatif konuma kalıcı restore
  • En yaygın kullanılan standart yöntem
03

Instant Recovery

  • Restore + Publish birleşimi, DB anında online
  • Switchover ile kalıcı hale gelir

Ekstra · Kurumsal Eklenti

SQL Plug-in ile Restore

Veeam Plug-in for Microsoft SQL Server, SQL Server üzerinde native backup/restore komutlarını kullanarak çalışır. Bütün veritabanını hızlıca orijinal ya da farklı bir SQL Server'a/instance'a geri yükler.

01

Latest State

  • En son restore point'e döner
  • Log backup yoksa varsayılan senaryo budur
02

Point-in-Time State

  • Transaction log backup'a dayanır
  • Belirli bir ana kadar geri döner
03

Başka Sunucuya

  • Farklı SQL Server / instance'a restore
  • Kimlik doğrulama: hesap bilgileri ya da recovery token
İki araçla çalıştırılır: GUI'de Restore Database Wizard, ya da komut satırında MSSQLRecoveryManager.exe.
